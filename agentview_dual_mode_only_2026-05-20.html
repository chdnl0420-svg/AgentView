요구사항 재진술

  • 카드 모드 — 지금과 동일. 위에 카드 격자, 아래에 새 작업 입력 바.
  • 단일화면 모드 — 왼쪽에 세션 목록 (가로 30%) + 오른쪽이 작업 창 (70%).
  • 왼쪽 상단 "＋ 새 작업 만들기" 버튼. 누르면 카드 모드의 입력창이 작업 창 가운데에 뜸.
  • 메시지 입력 → "새 작업 시작" → 세션 생성 → 자동 선택.
  • 세션 클릭 → 작업 창이 기존 카드 모드의 세션 상세창과 동일 구성.
  • 단일 모드의 빈 화면은 없음. 기본이 새 작업 입력 화면.

핵심 약속

  1. 카드 모드는 1줄도 안 건드린다 — 기존 사용자 충격 0.
  2. 새 컴포넌트 SessionList 하나 — 왼쪽 30% 폭의 줄 단위 목록.
  3. InputBar·SessionDetail 재사용 — 카드 모드와 동일 컴포넌트가 단일 모드 작업 창에서도 동작.
  4. 모드 토글 한 번 — 상단 chrome 우측 아이콘. localStorage 저장.
  5. 기본값: 카드 모드 — 처음 켜는 사용자에게 변화 없음.

단일 모드의 세 가지 상태

상태 1 — 기본 (세션 선택 X)

왼쪽에 세션 목록 + ＋ 버튼. 오른쪽 작업창 가운데에 새 작업 입력 패널이 자동 표시. 빈 화면 없음.

상태 2 — 세션 선택 후

세션 클릭하면 오른쪽 작업창이 카드 모드의 세션 상세화면과 동일해진다. 왼쪽 목록은 그대로 유지.

상태 3 — ＋ 다시 누르기 (선택 해제)

세션 보다가 ＋ 누르면 selectedId=null → 자동으로 상태 1 로 복귀. 작성 중이던 새 작업 입력 글은 newDraft 에 저장되어 있어 그대로 살아있음.

모드 전환 토글

  • 상단 chrome 우측에 작은 아이콘 두 개 (카드 ▣ / 단일화면 ▤).
  • 한 번 누르면 모드 전환 + localStorage 의 agentview.viewMode 갱신.
  • 다음 실행 때 마지막 모드 그대로 켜짐.
  • 처음 켜는 사용자 기본값: 카드 모드.

영향 사용자 시나리오

  • 기존 카드 모드 사용자: 변화 0. 상단에 토글 아이콘 두 개만 추가.
  • 여러 세션을 빠르게 오가는 사용자: 단일 모드 토글 → 왼쪽 목록에서 한 클릭으로 세션 전환. 카드 모드의 "뒤로 → 다른 카드 클릭" 두 단계가 한 단계로.
  • 새 작업 자주 시작하는 사용자: 단일 모드 기본 화면이 입력창. 매번 ＋ 누를 필요 없음.

마스터 계획과의 관계

  • 이 작업은 마스터 계획의 약속 1번만 떼어낸 것. 다른 약속 (백엔드·avd·Codex·권한 배지) 과 독립적으로 진행 가능.
  • 마스터의 Step E (20~22) 가 이 단독 계획과 동일.
  • 이 작업을 먼저 끝낸 뒤 나중에 마스터의 백엔드 셀렉트가 들어오면, SessionList 와 카드 모두 같은 백엔드 배지 컴포넌트를 추가하면 됨.
  • SessionDetail 의 권한 배지 개선도 마스터의 Step F 에서. 단일 모드의 작업창은 카드 모드의 SessionDetail 을 그대로 재사용하므로 자동 적용.
